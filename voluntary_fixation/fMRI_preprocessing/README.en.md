# StudyForrest fMRI preprocessing → SRM-input npz

*English version. 日本語版: [README.md](README.md).*

Pipeline that turns ds000113 (StudyForrest, `ses-movie/task-movie`, 8 runs) into denoised BOLD,
ROI intermediates, and the SRM-input npz files. It produces three things:

```
data/forrestgump/studyforrest/
  bold/       ← Step 1: denoised BOLD (sub-XX_run-Y_space-MNI152NLin2009cAsym_bold.nii.gz)
  bold_int/   ← intermediates auto-generated by Step 2 (subXX_base.bin / subXX_sideLR_roi44_all.bin ...)
  roi22LR_np/ ← final output of Step 2 (subXX_sideLR_roi44_data-12345678.npz)  ← SRM input
```

Whole pipeline:
```
BIDS raw (ds000113, ses-movie/task-movie, run 01..08)
  │  Step 0: fMRIPrep
  ▼  datasets_out/studyforrest-data-all_out/fmriprep-latest/fmriprep/.../*_desc-preproc_bold.nii.gz (+ confounds)
  │  Step 1: preprocessing_align_d_studyforrest.py
  ▼  bold/*_bold.nii.gz  (confound regression, band-pass, zscore applied)
  │  Step 2: prepare_npz.py   ← builds bold_int/ itself while writing the npz
  ▼  roi22LR_np/*.npz
```

Assumptions:
- Input task is `ses-movie/task-movie` (the audio-visual Forrest Gump, 8 runs), **not** `ses-forrestgump`
  (the audio-only version).
- Space is `MNI152NLin2009cAsym`. TR = 2.0 s. Subjects are the 12 with complete functional data:
  `01,02,03,04,06,10,14,15,16,17,18,19` (05, 09, 20 excluded).

## Step 0. fMRIPrep (BIDS raw → standard-space BOLD)

**Goal**: preprocess `ses-movie/task-movie` (8 runs) and obtain the MNI-space `desc-preproc_bold`
and confounds.

**Command**:
```bash
# If needed, fetch the actual data with DataLad first:
# cd /home/kazu/shared_response/datalad/DataLad-101/ds000113 && datalad get sub-*/ses-movie/func/*task-movie*

fmriprep-docker \
    /home/kazu/shared_response/datalad/DataLad-101/ds000113 \
    /home/kazu/shared_response/datasets_out/studyforrest-data-all_out/fmriprep-latest \
    participant \
    --participant-label 01 02 03 04 06 10 14 15 16 17 18 19 \
    --output-spaces MNI152NLin2009cAsym \
    --fs-license-file /path/to/license.txt
```

**Produces**: `*_space-MNI152NLin2009cAsym_desc-preproc_bold.nii.gz` and
`*_desc-confounds_timeseries.tsv` under
`datasets_out/studyforrest-data-all_out/fmriprep-latest/fmriprep/sub-XX/ses-movie/func/`.

> This repository does not include the fMRIPrep run script (run externally). Step 1 onward takes
> this output as input.

---

## Step 1. denoise → `bold/`

**Goal**: from the fMRIPrep output, regress out confounds, band-pass (0.00667–0.1 Hz), and zscore to
produce `bold/*.nii.gz` (96 files).
(Script: `preprocessing_align_d_studyforrest.py`; core: `preprocessing_utils.preprocess()`.)

**Command**:
```bash
python preprocessing_align_d_studyforrest.py
```

**Produces**: `bold/sub-XX_run-Y_space-MNI152NLin2009cAsym_bold.nii.gz` (12 subjects × 8 runs = 96 files).

---

## Step 2. npz-ify → `roi22LR_np/` (bold_int auto-generated)

**Goal**: mask `bold/` with HCP-MMP1 (left/right split, 44 ROIs), normalize per run × ROI, concatenate,
and write the SRM-input npz. Along the way it **builds `bold_int/` itself**
(`subXX_base.bin` / `subXX_sideLR_roi44_all.bin` / `_rois.bin`), reusing it if already present.
Script: `prepare_npz.py`.

**Command**:
```bash
python prepare_npz.py --side LR --n-roi 22    # left/right split, 44 ROIs → roi22LR_np/
# python prepare_npz.py --side '' --n-roi 22  # merged L/R, 22 ROIs → roi22_np/
```

**Produces**:
- `roi22LR_np/subXX_sideLR_roi44_data-12345678.npz` (12 subjects; contents: `roi_data, roi_name,
  n_scan_per_run, tr`)
- `bold_int/subXX_base.bin` and other intermediates (~12–15 min per subject on the first pass from `bold/`)

> Normalization: `prepare_npz.py:134` `if True :` (per-run z-score over time, per voxel) is the default.
> Switch it to `False` only if you want no normalization.

## (Optional) inter-ROI alignment analysis

`align_multiple_rois_studyforrest.py` is a **separate analysis** of inter-ROI alignment and is
**not needed** to produce `roi22LR_np` (Step 2 builds the `bold_int` intermediates itself). When run,
it creates `subXX_roi22_*` bin files under `bold_int/` and then runs ISC / alignment (`Pool`
parallelism, heatmap output). Only when needed:
```bash
python align_multiple_rois_studyforrest.py
```

## Path design (`paths.py`)

Every script reads data locations from `paths.py` — the single aggregation point that makes them
CWD-independent.

| Constant | Value | Purpose |
|---|---|---|
| `REPO_ROOT` | `/home/kazu/shared_response` | Parent of `data/` and `datasets_out/`. **On relocation, edit only this.** |
| `DATASETS_OUT` | `$REPO_ROOT/datasets_out` | Output root of Step 0 |
| `BOLD_DIR` | `.../studyforrest/bold` | Step 1 output / Step 2 input |
| `BOLD_INT_DIR` | `.../studyforrest/bold_int` | Step 2 intermediates |
| `npz_dir(n_roi, side)` | `.../studyforrest/roi{n_roi}{side}_np` | Step 2 npz output |

> `REPO_ROOT` here is the **data root** and must match `envs.py`'s `../../` (the two levels above the
> `submit_code2/` repository). `roi22LR_np/` produced here is exactly `envs.py`'s `BOLD44_ROOT`, the
> input to the downstream analyses. If you move the data, keep `paths.py` and `envs.py` in sync.

The atlas paths are resolved by `atlas_definition.py` relative to `__file__` (`atlas/` lives in this
directory).
